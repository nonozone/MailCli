# MailCLI v0.1 RC Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a credible `v0.1 RC` of MailCLI with stable core contracts, a release-ready OSS surface, and enough inbound/outbound capability for real agent workflows.

**Architecture:** Keep the existing layered split intact: `cmd` for contracts and orchestration, `driver` for transport, `parser` for normalization, and `composer` for outbound MIME. This phase is not about adding broad new surface area; it is about turning the current MVP into a stable, contributor-ready release candidate.

**Tech Stack:** Go, Cobra, enmime, go-imap, stdlib SMTP, YAML config, Python examples, GitHub docs.

---

## Scope

### Must complete for `v0.1 RC`

- release metadata and repository hygiene
- accurate docs aligned with actual implementation
- stronger outbound MIME support
- useful inbox query/list ergonomics
- safer config handling for real mailbox usage
- contributor-facing extension and governance docs

### Should complete if time allows

- better parser corpus and fixture breadth
- cleaner transport error typing
- initial CI automation

### Can defer until after `v0.1`

- local indexing and search
- full plugin runtime
- OAuth and provider-specific auth helpers
- advanced workflow automation commands
