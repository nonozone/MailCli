# MailCLI v0.1 RC 发布说明

## 摘要

`v0.1 RC` 标志着 MailCLI 第一次具备了“开放的 AI Native 邮件接口”雏形，而不再只是一个 parser 实验。

当前项目已经具备一条可用的端到端链路：

- 将本地邮件解析成 `StandardMessage`
- 从 IMAP 账户列出并抓取邮件
- 将近期邮件同步到本地索引并在本地检索
- 查看 thread 摘要和完整本地 thread，服务 agent 分拣
- 编译并发送出站草稿与回复
- 与 Python、shell 以及外部 agent provider 协作

## 当前范围

- parser-first 的入站标准化
- 基础 IMAP 读路径
- 本地 sync、search 和 thread 检索
- 面向 IMAP 风格账户的 SMTP 发信路径
- 稳定的机器接口 JSON 契约
- agent 示例和 provider handoff 示例
- 面向贡献者的扩展文档

## RC 之前已经补齐的关键能力

- `parse`、`list`、`get`、`send`、`reply` 命令集
- `sync`、`search`、`threads`、`thread` 本地检索闭环
- `DraftMessage`、`ReplyDraft`、`SendResult`
- 基于 `reply_to_message_id` 和 `reply_to_id` 的线程回复支持
- `multipart/alternative` 出站编译
- composer 附件打包
- 面向密码字段的环境变量秘密注入
- 更丰富的 inbox list 摘要，以及 mailbox/limit 控制
- 更清晰的 sync / index 计数输出和更丰富的 thread triage 字段
- 更强的 HTML 主体提取与追踪 URL 归一化
- external provider 契约与 OpenAI provider 示例
- 扩充后的 parser 样本集

## 建议视为稳定的集成边界

在 `v0.1 RC` 阶段，建议视为稳定边界的部分：

- `mailcli parse`
- `mailcli list`
- `mailcli get`
- `mailcli sync`
- `mailcli search`
- `mailcli threads`
- `mailcli thread`
- `mailcli send`
- `mailcli reply`
- `StandardMessage`
- `DraftMessage`
- `ReplyDraft`
- `SendResult`

## 仍属于 Heuristic 的部分

以下能力已经有用，但仍应视为持续演进：

- action 提取覆盖率
- 常见布局之外的验证码提取
- 异常 HTML 模板下的正文选择与清洗
- token 估算

## 本次 RC 明确不做的事

- 完整终端邮件客户端体验
- 在 core 中集成大量 OAuth 逻辑
- 运行时动态插件加载
- 基于 SQLite / FTS 的本地存储后端
- 在共享 parser / composer 层写 provider 私有业务规则

## RC 之后建议优先做的事

- 继续扩充 parser 样本集
- 补齐 parser onboarding 和 driver harness
- 提升本地 search 语义与排序
- 在共享契约更稳后增加更多 provider 实现
