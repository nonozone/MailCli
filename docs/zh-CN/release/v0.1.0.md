# MailCLI v0.1.0 发布说明

## 简介

`v0.1.0` 是 MailCLI 的第一个稳定版本。MailCLI 是一个 AI-native 的邮件接口，将原始 MIME 转换成适合 Agent 和 LLM 工作流消费的结构化上下文。

本版本包含完整的读-写-监控循环、高性能 SQLite + FTS5 本地索引、持久化 watch 状态，以及所有命令的稳定 JSON 契约。

---

## 本版本内容

### 新增命令（RC 之后）

| 命令 | 说明 |
|------|------|
| `mailcli watch` | 从一个或多个邮箱流式输出 JSONL 事件（IMAP IDLE + 轮询兜底）|
| `mailcli delete` | 从远端邮箱删除一封邮件 |
| `mailcli move` | 将邮件移动到另一个邮箱 |
| `mailcli mark` | 将邮件标记为已读或未读 |
| `mailcli export` | 将本地索引导出为 JSONL、JSON 或 CSV |
| `mailcli config show\|test` | 查看账户配置（密码已脱敏）或测试连接 |

### SQLite + FTS5 本地索引

本地索引已从 JSON 平面文件完全迁移到带 FTS5 全文搜索的 SQLite 数据库。

- **FTS5 召回** — unicode61 分词器，前缀搜索，忽略变音符号
- **字段权重重排序** — subject > snippet > body > sender（Go 层 `scoreMatch`）
- **BulkUpsert** — 所有消息在单个事务中写入，比逐条提交快 **3.2 倍**
- **BulkHas** — 单次 SQL 查询检查 N 个 ID（替代 N 次往返）
- **WAL 模式** — 写入期间允许并发读取
- **向后兼容** — `.json` 路径自动静默映射到 `.db`

### 持久化 Watch 状态

`mailcli watch --index <path>` 现在在 SQLite 中维护一张 `watch_seen` 表。

- 启动时从数据库恢复已 seen 的 ID
- 进程重启后不重复发送任何消息
- 首次运行时静默写入当前邮箱内容（不会产生旧邮件洪流）
- `--auto-sync` 在同一事务中将新消息写入本地索引

### AI 工具 Schema

可直接用于 LLM 函数调用的 JSON Schema：

- `tools/openai.json` — OpenAI Function Calling 格式（12 个工具）
- `tools/anthropic.json` — Anthropic Tool Use 格式（12 个工具）
- `tools/agent_example.py` — 参考实现：`watch → LLM → reply` 管道
- `tools/README.md` — 集成指南，含 `tool_to_cmd` 分发模式

### Parser 改进

- 中文邮件 fixture 覆盖：验证登录、重置密码、账单、退订、确认订阅、附件、在线查看、滥用举报
- 全角数字验证码提取
- Safe-link URL 还原（Proofpoint、Barracuda）
- 确认邮件启发式规则收紧
- 确认订阅关键词覆盖扩大

### 示例与 Fixture

- `examples/python/agent_thread_assistant.py` — 线程模式 AI Agent，支持外部 provider
- `examples/python/agent_inbox_assistant.py` — 收件箱模式 AI Agent
- `examples/providers/` — 模板 provider 和 OpenAI 外部 provider 实现
- `examples/artifacts/local-thread-demo/` — 完整本地线程 demo 制品

## 从 RC 升级

无需迁移。已有的 `*.json` 索引文件在首次使用时会自动映射到 `*.db`，并创建新的 SQLite 数据库。

```bash
# 现有用法无需任何修改：
mailcli sync --index /tmp/mailcli.json  # → 透明地使用 /tmp/mailcli.db
```

## 快速开始

```bash
go build -o mailcli ./cmd/mailcli

# 零网络快速体验
./mailcli sync --config examples/config/fixtures-dir.yaml \
  --account fixtures --index /tmp/demo.db --limit 0
./mailcli threads --index /tmp/demo.db

# Watch + AI Agent
mailcli watch --account work --index ~/.config/mailcli/index.db --auto-sync \
  | python3 tools/agent_example.py
```
